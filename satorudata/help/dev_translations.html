Reggie! Translations

This page will explain how to create Reggie! Translations plug-ins. Each translation consists of multiple files containing text which the editor uses instead of the default versions.

How Hard will it Be?

Translations are relatively easy to make because the file formats you will work with are fairly simple. However, these files are very large and it will take you several hours to translate the entire editor. Plan to spend a few days to a week, depending on how much time you can spend on it each day.

In addition, you need to know what XML files are and how to make them. If you're not familiar with that, Google it and then continue.

Getting Started: Creating main.xml

Each Reggie! translation is located within its own folder in reggiedata/translations/. The name of the folder doesn't matter, but make it something unique and descriptive.

Inside that folder, create a file named main.xml. Open it with a plain-text editor or XML editor and add the following code:

<?xml version="1.0"?>
<translation name="" version="1.0" translator="">
    <strings path="strings.xml" />
    <file name="bga" path="bga.txt" />
    <file name="bgb" path="bgb.txt" />
    <file name="entrancetypes" path="entrancetypes.txt" />
    <file name="levelnames" path="levelnames.xml" />
    <file name="spritecategories" path="spritecategories.xml" />
    <file name="spritedata" path="spritedata.xml" />
    <file name="tilesets" path="tilesets.xml" />
    <file name="ts1_descriptions" path="ts1_descriptions.txt" />
</translation>

Add the name of the language between the quotes of the name attribute in the <translation> tag and add your name between the quotes of the translator attribute.

Adding Files

The main.xml you've created instructs Reggie! to load various files from your translation's folder. Since those files don't exist there yet, you need to make copies there. Copy the following files to your game patch folder:

  • reggiedata/translations/strings.xml
  • reggiedata/bga.txt
  • reggiedata/bgb.txt
  • reggiedata/entrancetypes.txt
  • reggiedata/levelnames.xml
  • reggiedata/spritecategories.xml
  • reggiedata/spritedata.xml
  • reggiedata/tilesets.xml
  • reggiedata/ts1_descriptions.txt

After copying each of these to your translation's folder, translate them one by one. Each has a different format; open each one with a plain-text editor and see how it works. strings.xml and spritedata.xml are by far the largest files, so you may want to begin with those.

Explanation of Strings

If you're not a programmer, you may have no idea what is meant by "strings." "String" is the technical term for a chunk of text, which could be anything from a single letter to an entire paragraph. To change the rest of the text in Reggie!, you need to edit strings.xml. strings.xml contains all of the miscellaneous strings in the editor that don't fit anywhere else.
