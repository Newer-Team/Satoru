Reggie! Themes

This page will explain how to create Reggie! Themes plug-ns. Minimally, themes define how in-editor objects are colored. They can also contain a custom icon set and change the UI style.

How Hard will it be?

Themes are the easiest type of Reggie! plug-in to create. You only need to work with one XML file unless you plan to add custom icons. Before starting, though, you need to know what XML files are and how to make them. If you're not familiar with that, Google it and then continue.

Getting Started: Creating main.xml

Firstly, every Reggie! theme must be located within reggiedata/themes. To create a new theme, make an empty XML file in that folder. You can name it anything as long as it ends with ".xml". Open it with any plain-text editor or XML editor.

Setting Up main.xml

After opening the file, add the following code:

<?xml version="1.0"?>
<theme name="" version="1.0" creator="" description="">
</theme>

Now you need to fill in the empty attributes. Inside name="", insert the theme name. Add your username as the value of creator, and describe your theme in description.

Adding Colors

Now we need to add colors to the theme. Colors are defined between the <theme> and </theme> tags as <color /> tags. Here's the format of those:

<color id="value" r="value" g="value" b="value" a="value" />

To create a color tag, you need to know specifically which color you want to change. At the end of this page is a Theme Color Reference; use that to find valid IDs. IDs are case-insensitive, but the recommended capitalization is to use completely lower case letters.

After you've picked an ID, you need to decide on a new color to change it to. There are many ways of selecting an RGB (red-green-blue) color; Google it and find a website that lets you try different colors. Each value must be a decimal integer between 0 and 255. Once you have R, G, and B color values, you can decide on an A value. The A value determines the opacity of the color: a="255" would produce a fully opaque color whereas a="0" would produce a fully transparent color. Keep in mind that the A value is not required, but if you do not add it, Reggie! will use whatever A value the original color had.

You can add as many colors as you want, but please follow two guidelines:

  1. Don't repeat colors. There's no reason at all to do this. If you do, Reggie! will pick whichever color comes last.
  2. Don't add entries for colors that you have not changed. Colors that are not defined within your theme XML will automatically switch to the Classic theme. Try to keep the size of your theme XML down to a minimum by removing unnecessary data such as this.

Another thing you should know is that Reggie! can "guess" certain colors. You see, in the editor, certain colors are often identical. If you define one color in your theme but not another, Reggie! will assume that you want both of them to have the same value if they're usually identical. For example, if you define location_fill but not overview_location_fill, Reggie! will automatically set overview_location_fill to the new value of location_fill, rather than use the value defined by the Classic theme. In the Theme Color Reference at the bottom of this page, the "Defaults To..." column shows the color Reggie! will use if a color is not defined.

Adding Icons

In addition to colors, Reggie! themes also support custom icons. While you can change the entire icon set, most themes with custom icons only change a few. No matter how many icons you change, the process is the same. Follow the steps below if you would like to do this:

  • The first step is to add the following attribute to the theme tag of your XML:
    icons="True"
    "True" is case-insensitive, but icons is not. It does not matter where you put the attribute, as long as it is in the <theme> tag.
  • Next, you need to do create a folder to store your custom icons in. Here are the two requirements:
    1. The folder must be located in reggiedata/themes/
    2. The folder must be named XMLFileName_ico where XMLFileName is the filename of your XML file, without the .xml extention.
    If you do not follow these, Reggie! will not be able to find the icons, and they will not be used.
  • Finally, you need to have some icons to put in that folder. Go to reggiedata/ico/ . All of Reggie's default icons are stored within that folder. If you would like to modify them in your theme, copy-and-paste some into your new XMLFileName_ico folder, and edit them with any image editor. Instead, you could also make icons from scratch or download some free ones from the Internet. In order for Reggie! to recognize them, though, they must have the exact same filename as the icon they are replacing in reggiedata/themes/ !

Changing the UI Style

Reggie! Themes can also change the UI style. This feature dramatically alters the feel of the editor. To edit the style, add the following attribute to the <theme> tag in your theme XML:

ui=""

Add the name of a style between the quotes. Different operating systems support different styles, but here's a list of styles available on most computers:

  • windows
  • motif
  • cde
  • plastique
  • cleanlooks
